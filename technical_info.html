Specifications

Secure storage and secure transmission of data integrates many ideas which often need to composed separately in different applications. Data provenance capabilities add theneed for further composable components. We make an effort to breakout ideas into independent specifications so that they can be integrated as appropriate for different projects.

BluMamba Patent
BluMamba is the codename assigned to the conecpt of cryptographically dividing data into shares and managing the state transitions necessary for data disassembly and data reassmebly. This aim to distribute these shares across different geographical locations.
PinkScorpion Protocol on a Parachain
Explore this resource for details on the parachain (blockchain) implementation responsible for executing the state transitions necessary for data disassembly and reassmebly.
PinkScorpion Patent
PinkScorpion is the codename for the data disassembly and reassmebly protocol, supported by a state machine operated by a blockchain. The blockchain serves as evidence of the integrity of the data disassembly process and any subsequent reassmebly process.
Filesystem Overview (Small and Large Objects)
This paper discusses the NFS-based filesystem designed for efficient and secure storage, with the entire access lifecycle recorded on an immutable log.
Data Integrity Whitepaper
This paper provides an overview of the concepts surrounding data integrity, chain of custody, and provenance. These terms are commonly utilised across different domains and scenarios and are specifically addressed by Lockular components and products.
Cloud native IDE playground (Secure Working)
This paper outlines a versatile cloud-based design environment supported by Lockular's filesystem technology. The IDE is built on VSCode server and the paper elaborates on how various design toolchains can be loaded and utilised. Throughout the process, data remains protected and its usage is meticulously tracked.
